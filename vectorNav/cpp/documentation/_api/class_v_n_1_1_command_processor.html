class CommandProcessor¶

The CommandProcessor class is the handshake between the user-facing code (such as Sensor) and the communication code interfacing with the sensor (such as Serial and PacketProcessor). Consequently, it exists on multiple threads (all exists on the main thread except for matchResponse, which is called from the high-priority thread) with an internal queue to handle the cross-thread communication. Its chief responsibility is to facilitate the communication of commands between the user and the sensor, tracking received responses (sent via matchResponse) and correlating them with known-sent commands.
